# 风格 E：Apple 高级克制风 (Apple Refined)

> 设计关键词：**极致留白、字体即设计、重力感排版、无装饰主义、呼吸节奏**

---

## 设计理念

Apple.com 的设计哲学只有一句话：**"Remove everything that is not essential."** 这份简历追求的不是"好看"，而是"高级"。没有色条、没有标签、没有卡片、没有任何"设计元素"——所有的视觉张力完全来自于**字体大小的对比**、**字重的层级**和**大面积的留白**。就像 Apple 官网上，一行大字 + 一片空白，就足以让你停下来看。

**核心法则**：当你想加一个装饰元素的时候，停下来，删掉它。

---

## 视觉规范

### 配色方案
| 用途 | 色值 | 说明 |
|------|------|------|
| 页面背景 | `#ffffff` | 纯白，没有任何妥协 |
| 主文字 | `#1d1d1f` | Apple 标志性的近黑色（不是 `#000`） |
| 次文字 | `#86868b` | Apple 中灰，大量使用 |
| 弱文字 | `#afafaf` | 浅灰，用于最次要信息 |
| 链接/极少量强调 | `#0071e3` | Apple 蓝，全简历仅出现 1-2 处 |
| 分割线 | `#d2d2d7` | Apple 标志性的浅灰线 |
| 背景（无） | — | 不使用任何有色背景块 |

> **注意**：这套配色只有 **黑、灰、白** 三个层级 + 一个几乎不用的蓝色。没有"主题色"概念。克制本身就是主题。

### 字体
```css
font-family: -apple-system, 'SF Pro Display', 'SF Pro Text', 'PingFang SC', 
             'Helvetica Neue', 'Noto Sans SC', sans-serif;
```
> SF Pro 是 Apple 的系统字体。Display 变体用于大标题，Text 变体用于正文。在非 Apple 设备上 fallback 到 Helvetica Neue。

**字体层级是这套风格的灵魂：**

| 元素 | 字号 | 字重 | 颜色 | 字距 | 说明 |
|------|------|------|------|------|------|
| 姓名 | `40px` | `700` | `#1d1d1f` | `-1px` | 大、重、紧凑，像产品发布会的标题 |
| 求职意向 | `20px` | `600` | `#1d1d1f` | `-0.3px` | 紧跟姓名，像副标题 |
| Section 标题 | `24px` | `600` | `#1d1d1f` | `-0.3px` | 大号中文，不加英文，不大写 |
| 公司/项目名 | `17px` | `600` | `#1d1d1f` | `0` | |
| 职位/角色 | `15px` | `400` | `#86868b` | `0` | 灰色，弱化 |
| 正文 | `14px` | `400` | `#1d1d1f` | `0` | `line-height: 1.65` |
| 日期 | `14px` | `400` | `#86868b` | `0` | |
| 项目背景 | `14px` | `400` | `#86868b` | `0` | 灰色正文 |
| 联系信息 | `13px` | `400` | `#86868b` | `0.2px` | |

> **关键特征**：标题用 **负字距**（letter-spacing 为负值），这是 Apple 排版的标志性手法，让大字看起来更紧凑、更有重力感。

### 布局
- **整体**：单栏，A4 尺寸
- **页边距**：上 `32mm`，下 `24mm`，左右 `28mm`（比其他风格更大的边距 = 更多留白）
- **Section 间距**：`36px`（极大，制造呼吸感）
- **条目间距**：`24px`
- **行高**：正文 `1.65`，标题 `1.2`

### 核心视觉元素

1. **Header 区 —— 产品发布会式开场**
   - 姓名：40px，`font-weight: 700`，`letter-spacing: -1px`
   - 求职意向紧跟下方，20px，`font-weight: 600`，同色 `#1d1d1f`
   - 两者之间 `4px` 间距，像 Apple 产品页的"iPhone 16 Pro" + "钛金属强大如斯"
   - 联系信息在求职意向下方 `16px`，一行排列，`font-size: 13px`，`color: #86868b`，用 `  |  ` 分隔（两侧各一个空格的竖线）
   - Header 区底部 **没有分割线**，靠 `48px` 的纯留白与正文拉开距离
   ```
   何伟杰                          ← 40px, 黑, 粗
   AI Agent 应用研发工程师            ← 20px, 黑, 半粗
   
   13544248446  |  heweijie0306@gmail.com  |  GitHub  |  深圳   ← 13px, 灰
   
   
                                    ← 48px 纯白留白
   ```

2. **个人总结 —— 无引号、无边框、纯文字**
   - 不使用任何引用样式（无 border-left、无背景色、无引号）
   - 直接用 `font-size: 15px`，`color: #1d1d1f`，`line-height: 1.7` 写一段话
   - 和正文的区别仅在于**字号略大**（15px vs 14px）
   - 段落下方 `36px` 留白

3. **Section 标题 —— 大号中文，仅此而已**
   - 只写中文（如「工作经历」），不写英文
   - `font-size: 24px`，`font-weight: 600`，`color: #1d1d1f`
   - `letter-spacing: -0.3px`
   - 标题下方 `16px` 后直接接内容
   - **没有下划线、没有色条、没有任何装饰**
   - 靠字号本身（24px vs 正文 14px）制造层级
   ```
   工作经历                         ← 24px, 就这样, 什么都不加
   ```

4. **公司/项目行**
   - 公司名 `17px`，`font-weight: 600`，`color: #1d1d1f`
   - 职位在公司名同一行右侧，或换行，`15px`，`color: #86868b`
   - 时间：`14px`，`color: #86868b`，右对齐或紧跟职位
   - 公司名与职位之间用 `—`（em dash）分隔，不用 `|`

5. **正文 bullet points —— 无 bullet**
   - **不使用任何 bullet 符号**（无 `•`、无 `▸`、无 `—`）
   - 每条经历直接以文字开头，靠 `margin-bottom: 8px` 区分条目
   - 如果一定要有视觉区分，用 `padding-left: 0`（不缩进），每条之间靠间距分割
   - 或者用极克制的方式：第一个词加粗作为"视觉锚点"
   ```
   主导 HTML 直出架构重构，废弃结构化模板约束...
   
   攻坚编辑态痛点，基于 DOM Parser 实现前端后处理...
   
   推动 MVP 闭环，利用业余时间独立完成开发...
   ```

6. **技能展示 —— 纯段落式**
   - **不使用任何标签/badge/pill**
   - 按类别分行，每行是一句话式的描述
   - 技术名称用 `font-weight: 600` 加粗，不改变颜色
   - 像 Apple 产品页的规格参数区一样，平铺直叙

7. **加粗文字**
   - `font-weight: 600`，颜色不变（保持 `#1d1d1f`）
   - Apple 的设计里几乎不用"颜色"来强调，只用字重

8. **分割线（极少量使用）**
   - 如果 section 之间需要视觉分隔，使用 `1px solid #d2d2d7`
   - 但优先考虑**只用留白**，不用线
   - 全简历最多 1-2 条分割线

### 装饰元素
- ❌ 无色条 / border-left
- ❌ 无背景色块
- ❌ 无标签 / badge / pill
- ❌ 无图标 / icon
- ❌ 无阴影
- ❌ 无圆角卡片
- ❌ 无渐变
- ❌ 无 bullet 符号
- ❌ 无英文大写 section 标题
- ✅ 唯一的"设计"：字号对比 + 字重层级 + 留白节奏

### 空间节奏（The Rhythm）
这是 Apple 风格最核心的设计要素——**间距本身就是设计**：

```
[32mm 顶部留白]
姓名 (40px)
[4px]
求职意向 (20px)
[16px]
联系信息 (13px, 灰)
[48px ← 大留白]
────────────────── (可选分割线)
[36px]
个人总结 (15px)
[36px ← 大留白]
工作经历 (24px)
[16px]
  公司名 (17px)
  [4px]
  职位 — 时间 (15px, 灰)
  [12px]
  正文第一条 (14px)
  [8px]
  正文第二条 (14px)
[24px]
  公司名...
[36px ← section 间大留白]
项目经历 (24px)
[16px]
  项目名 (17px)
  ...
[24mm 底部留白]
```

### 打印适配
```css
@media print {
  /* Apple 风格本身就是白底黑字，打印几乎无需调整 */
  body { margin: 0; }
  a { color: #1d1d1f; text-decoration: none; }
}
```

## 整体气质描述

> 如果这份简历是一个人，他穿的是：**白色 T 恤 + 深灰色休闲裤 + 白色 Air Max**。没有 logo，没有配饰，但面料一摸就知道不便宜。
> 
> 他不需要用颜色、标签、色条来"装饰"自己。他的简历像 Apple Store 里的展示台——大面积留白之中，产品（内容）自己会说话。

## ⚠️ 适用场景提示

**最适合**：
- 任何场景（这是 ATS 最友好的风格之一，纯文字 + 语义化 HTML）
- 想让内容本身成为焦点的人
- 审美自信、不需要靠设计元素来"显得专业"的候选人
- 追求"less is more"极致体验的场景

**风险提示**：
- 如果简历内容本身不够强，大面积留白会放大内容的单薄感
- 对内容质量要求最高——没有视觉元素"帮忙"，文字必须字字珠玑
- 可能被不熟悉这种审美的 HR 误认为"排版太简单了"（但懂的人会懂）

## 设计哲学总结

> **Vercel 风格说的是"我是技术前沿的人"。**
> **Apple 风格说的是"我的内容好到不需要任何装饰"。**
> 
> 两者都是高级，但 Apple 是那种更深层的高级——是减法做到极致之后的自信。
